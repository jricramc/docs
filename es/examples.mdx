## Ejemplos de plataformas

Mira lo que puedes construir con Nerd. Estos son prompts reales que crean plataformas completas y funcionales.

## Plataformas de e-commerce

<AccordionGroup>
  <Accordion title="Boutique de moda" icon="shirt">
    **Prompt:**
    ```
    "Crea una boutique en línea para ropa de mujer. Incluye categorías de 
    productos, filtros de talla, lista de deseos, carrito de compras y 
    checkout. Agrega un panel de administración para gestionar inventario 
    y pedidos."
    ```
    
    **Lo que incluye:**
    - Catálogo de productos con filtrado
    - Carrito de compras y checkout
    - Cuentas de cliente y listas de deseos
    - Dashboard de administración para inventario
    - Sistema de gestión de pedidos
  </Accordion>
  
  <Accordion title="Marketplace digital" icon="download">
    **Prompt:**
    ```
    "Construye un marketplace donde creadores puedan vender productos digitales. 
    Los vendedores suben archivos, establecen precios y gestionan ventas. Los 
    compradores pueden previsualizar, comprar y descargar. La plataforma toma 
    10% de comisión."
    ```
    
    **Lo que incluye:**
    - Soporte multi-vendedor
    - Carga y gestión de archivos
    - Funcionalidad de vista previa
    - Sistema de descarga seguro
    - Pagos basados en comisión
  </Accordion>
  
  <Accordion title="Servicio de caja de suscripción" icon="box">
    **Prompt:**
    ```
    "Crea un servicio de suscripción para cajas mensuales de snacks. Los clientes 
    pueden elegir tamaño de caja, establecer preferencias y gestionar suscripciones. 
    Incluye pagos recurrentes y un programa de referidos."
    ```
    
    **Lo que incluye:**
    - Gestión de suscripciones
    - Facturación recurrente
    - Preferencias de cliente
    - Sistema de referidos
    - Integración de envíos
  </Accordion>
</AccordionGroup>

## Consejos para construcciones exitosas

<CardGroup cols={2}>
  <Card title="Empieza con características principales" icon="seedling">
    Construye primero la funcionalidad esencial, luego agrega extras
  </Card>
  
  <Card title="Enfócate en funcionalidad" icon="gears">
    Describe qué deben poder hacer los usuarios, no cómo construirlo
  </Card>
  
  <Card title="Piensa en roles de usuario" icon="users">
    Define claramente quién usa la plataforma y qué pueden hacer
  </Card>
  
  <Card title="Itera mientras avanzas" icon="arrows-rotate">
    Refina y agrega características después de ver la construcción inicial
  </Card>
</CardGroup>
